# 1. Distinct Search Queries After January 18, 2024

## Question

List all distinct search queries made on or after January 18, 2024, ordered alphabetically.

## SQL Query

```sql
SELECT DISTINCT s.query_text AS query_text
FROM search_queries s
WHERE s.search_date >= '2024-01-18'
GROUP BY s.query_text
ORDER BY s.query_text;
```

## Answer / Result

Returns all unique search queries made on or after January 18, 2024.

## Insight

This query helps identify recent search trends and user interests on the platform.

# 3. Users Who Never Made a Search Query

## Question

Find users who have never made a search query.

## SQL Query

```sql
SELECT u.user_id AS user_id
FROM users u
LEFT JOIN search_queries s
ON u.user_id = s.user_id
WHERE s.user_id IS NULL
GROUP BY u.user_id
ORDER BY u.user_id;
```

## Answer / Result

Returns users who have never performed a search query.

## Insight

This query helps identify inactive or less engaged users.

# 11. Users Who Searched on Consecutive Days

## Question

Calculate users who searched on consecutive days.

## SQL Query

```sql
SELECT DISTINCT s1.user_id AS user_id
FROM search_queries s1
INNER JOIN search_queries s2
ON s1.user_id = s2.user_id
AND s2.search_date = DATE(s1.search_date, '+1 day')
GROUP BY s1.user_id
ORDER BY s1.user_id;
```

## Answer / Result

Returns users who performed searches on consecutive days.

## Insight

This query helps measure consistent user engagement and activity patterns.

# 12. Most Searched Query for Each Day

## Question

Find the most searched query for each day.

## SQL Query

```sql
SELECT ranked_queries.search_date AS search_date,
ranked_queries.query_text AS query_text
FROM (
SELECT s.search_date AS search_date,
s.query_text AS query_text,
COUNT(*) AS total_searches,
RANK() OVER (
PARTITION BY s.search_date
ORDER BY COUNT(*) DESC
) AS ranking
FROM search_queries s
GROUP BY s.search_date, s.query_text
) ranked_queries
WHERE ranked_queries.ranking = 1
ORDER BY ranked_queries.search_date;
```

## Answer / Result

Returns the most searched query for each day.

## Insight

This query identifies daily trending search topics on the platform.

# 13. User Contribution to Total Ad Revenue

## Question

Calculate each user's percentage contribution to total ad revenue.

## SQL Query

```sql
SELECT a.user_id AS user_id,
ROUND(
100.0 * SUM(a.revenue) /
(
SELECT SUM(revenue)
FROM ad_clicks
), 2
) AS revenue_percentage
FROM ad_clicks a
GROUP BY a.user_id
ORDER BY revenue_percentage DESC;
```

## Answer / Result

Returns each user's percentage contribution toward total ad revenue.

## Insight

This query helps identify the most valuable users from an advertising revenue perspective.

# 15. Top 10 Users by Revenue with Search Activity

## Question

Find top 10 users by revenue with their search activity and country.

## SQL Query

```sql
SELECT u.user_id AS user_id,
u.country AS country,
COUNT(DISTINCT s.search_date) AS search_days,
SUM(a.revenue) AS total_revenue
FROM users u
INNER JOIN search_queries s
ON u.user_id = s.user_id
INNER JOIN ad_clicks a
ON u.user_id = a.user_id
GROUP BY u.user_id, u.country
ORDER BY total_revenue DESC
LIMIT 10;
```

## Answer / Result

Returns the top 10 revenue-generating users along with their country and search activity.

## Insight

This query combines engagement and monetization metrics to identify highly valuable platform users.

# 16. Most Watched Video Categories

## Question

Find the most watched video categories based on total watch time.

## SQL Query

```sql
SELECT v.category AS video_category,
SUM(vw.watch_time) AS total_watch_time
FROM videos v
INNER JOIN video_views vw
ON v.video_id = vw.video_id
GROUP BY v.category
ORDER BY total_watch_time DESC;
```

## Answer / Result

Returns total watch time accumulated for each video category.

## Insight

This query helps identify the most engaging content categories and user viewing preferences on the platform.
